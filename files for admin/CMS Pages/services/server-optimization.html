1. Analysis

The first step is to perform an analysis of your current server setup. This can help identify potential bottlenecks or weak points affecting your store's performance. Here are the stages of the analysis:
  • Run Lighthouse test/PageSpeed Insights Tools
  • Run Pingdom Tools test
  • Run GTmetrix test
  • Review New Relic Report and identify slow pages
  • Blackfire analysis of slow DB queries
  • Compare data from all of the tests and create an overall roadmap of Magento performance

2. Must have Performance Tunings

Reviewing specific parameters to ensure the best server configuration settings is essential. Make sure you check and adjust each parameter as necessary to get the best performance from your server
  • Check if HTTP/2 or 3 is enabled
  • Check CDN (Content Delivery Network) settings
  • Check all Magento performance settings
  • Make sure Production Mode is on
  • Enable gzip compression
  • Ensure Lazy loading of images is implemented
  • Check if Asynchronous Indexing is enabled
  • Ensure Minify CSS/JS is enabled
  • Ensure Static Files are signed
  • Make sure all indexers are set to "Update on Schedule "
  • Check for and disable unused extensions
  • Install the latest supported PHP and MySQL version
  • Check if the webserver is running on NGINX PHP-FPM
  • Check "warm-up" / ensure cache warmer is installed
  • Ensure WebP is used instead of JPG or PNG
  • Use Varnish with Full Page Cache
  • Ensure OPcache is enabled and adequately tuned
  • Set Browser cache max-age at least to 7 days
  • Check all vital PHP settings
  • Ensure all sessions are stored in Redis

3. Database Tuning

Optimizing your database is a great way to reduce server load. You can check the database's size and modify indexes as needed. We recommend analyzing the database setup and configuration as follows
  • Analyze database size and Optimize it
  • Make sure that indexing does not lock the database tables
  • Check DB settings and ensure the DB cache is enabled and properly configured

4. Code Tuning

Code review may make your system perform better. A deeper look into each element of your existing codebase can reveal areas of improvement. This stage includes updating the latest Magento version and checking all custom integrations.
  • Create a list of long/chained requests for Code refactoring
  • Code review of custom integrations and features
  • Ensure coding standards and best practices are followed
  • Check for Magento extensions updates
  • Update Magento to the latest version
  • Configure pipeline deployment to reduce downtime for production on updates

5. Core Web Vitals

Core Web Vitals is a set of metrics essential for providing the best user experience. We'll help your website to gain the highest possible result in the green zone. Core Web Vitals measure website load time, interactivity, and visual stability. The leading indicators are:
  • Largest Contentful Point (LCP)
  • Cumulative Layout Shift (CLS)
  • First Input Delay (FID)
  • First Contentful Paint (FCP)
  • Interaction to Next Paint (INP)
  • Time to First Bite (TTFB).
